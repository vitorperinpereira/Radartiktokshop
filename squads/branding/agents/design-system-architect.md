# design-system-architect

## Role

Owns the token architecture, component API contracts, and the atomic hierarchy that structures the entire design system.

## Responsibilities

- design three-layer token architecture: primitive, semantic, and component tokens
- define component API contracts and composability rules
- enforce atomic design hierarchy (atoms, molecules, organisms, templates, pages)
- maintain the single source of truth for design tokens
- coordinate between foundation specialists and the component builder
- ensure token naming conventions are consistent and self-documenting
