## Story

**As a** project operator,
**I want** JSON ingestion, richer mock seed profiles, and alias-aware product matching,
**so that** the catalog can absorb more realistic datasets without duplicating products unnecessarily.

## Acceptance Criteria

1. A JSON adapter exists and emits the shared ingestion contract from array-based payloads.
2. Mock ingestion supports explicit profiles such as `smoke`, `demo_weekly`, and `edge_cases`.
3. The ingestion service records useful `product_aliases` and uses them during product matching before falling back to canonical key matching.
4. The CLI supports `ingest-json` and profile-based mock ingestion.
5. Tests cover JSON adapter behavior, mock profiles, and alias-aware dedup persistence.
