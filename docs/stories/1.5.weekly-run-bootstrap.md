## Story

**As a** project operator,
**I want** the weekly CLI run to persist a `pipeline_run` and generate deterministic product signals,
**so that** the scoring pipeline has a real execution record and reusable normalized inputs before agent scoring begins.

## Acceptance Criteria

1. A deterministic feature extraction worker exists and emits `product_signals` from the latest product snapshots.
2. A weekly orchestration service persists `pipeline_runs` with `running`, `completed`, or `failed` status transitions plus `input_job_ids`.
3. The CLI `weekly-run` command executes the bootstrap flow end-to-end and returns a structured JSON summary.
4. Tests cover signal extraction, pipeline run persistence, and the CLI weekly-run flow against sqlite.
