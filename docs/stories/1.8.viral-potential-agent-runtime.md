## Story

**As a** project operator,
**I want** a working Viral Potential Agent runtime on top of the shared scoring contracts,
**so that** the weekly pipeline can produce a second structured agent score before final aggregation is introduced.

## Acceptance Criteria

1. The shared scoring contract supports the metadata needed by the Viral Potential Agent.
2. A deterministic Viral Potential Agent runtime exists and returns a normalized score plus structured reasoning.
3. The weekly LangGraph flow executes the Viral Potential Agent after the Trend Agent and exposes the count of scored products in the run summary.
4. Tests cover Viral Potential scoring behavior and weekly-run integration with the Viral Potential Agent enabled.
