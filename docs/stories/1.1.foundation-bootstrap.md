## Story

**As a** project operator,
**I want** a runnable CLI-first project skeleton with local infrastructure and quality wrappers,
**so that** implementation can start from a stable, testable foundation.

## Acceptance Criteria

1. The repository contains Python project metadata and the base structure for `bin`, `apps`, `services`, `infra`, `tests`, and `scripts`.
2. A CLI entrypoint exists and can run a local health command without requiring the API or dashboard to be open.
3. A FastAPI app exposes a `/health` endpoint and a Streamlit placeholder exists as a read-only observability surface.
4. A Docker Compose file for PostgreSQL and a project-specific `.env.example` are present.
5. A `package.json` exposes `lint`, `typecheck`, `test`, and `build` wrappers aligned with the AIOX quality gates.
6. This story tracks implementation tasks, file list, and verification gaps.

## Dev Notes

### Implementation Context

- The architecture baseline in `docs/architecture.md` defines the product as a Python modular monolith with three surfaces: CLI, FastAPI, and Streamlit.
- `docs/setup.md` fixes the first stack cut as Python `3.12`, `uv`, FastAPI, Streamlit, PostgreSQL, SQLAlchemy or SQLModel, Alembic, LangGraph, pytest, ruff, and mypy or pyright.
- `docs/prd.md` and `docs/implementation-plan.md` require CLI-first operation before UI polish. The dashboard must remain read-only.
- `docs/roadmap.md` places this story in the foundation phase, before ingestion, scoring, and weekly reporting.

### Relevant Source Tree

- `bin/radar.py`: primary CLI control surface
- `apps/api`: HTTP surface for health and later read endpoints
- `apps/dashboard`: Streamlit observability surface
- `services/shared`: shared config and app metadata
- `infra/docker`: local database runtime
- `infra/migrations`: migration placeholders for the next story

### Constraints

- Do not depend on real TikTok Shop connectors in this story.
- Keep deterministic setup logic separate from future runtime agent logic.
- Focus on local developer bootstrap and minimal runnable surfaces only.

### Testing

- Lightweight checks for this story should prefer syntax and local bootstrap validation.
- `python bin/radar.py health` should work without third-party services running.
- Python files should at least pass syntax compilation.
- Full `lint`, `typecheck`, `test`, and `build` wrappers should exist even if dependencies are not installed yet.
