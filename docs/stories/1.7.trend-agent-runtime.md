## Story

**As a** project operator,
**I want** a shared agent scoring contract and a working Trend Agent runtime,
**so that** the weekly pipeline can produce the first structured agent score and reasoning payloads from persisted product signals.

## Acceptance Criteria

1. Shared scoring contract primitives exist for MVP analysis agents.
2. A deterministic Trend Agent runtime exists and returns a normalized score plus structured reasoning.
3. The weekly LangGraph flow executes the Trend Agent after deterministic feature extraction and exposes the count of scored products in the run summary.
4. Tests cover contract serialization, Trend Agent scoring behavior, and weekly-run integration with the Trend Agent enabled.

## Dev Notes

### Implementation Context

- Story 1.6 already introduced the LangGraph weekly orchestration layer and typed weekly state.
- `docs/scoring-model.md` defines `Trend Agent` as a normalized `0-100` score with structured reasoning.
- `docs/agents.md` defines runtime agents as interpretation layers on top of deterministic signals.

### Constraints

- Keep this story limited to the Trend Agent only.
- Do not implement Viral Potential, Creator Accessibility, or final score aggregation here.
- Preserve deterministic behavior for now; AI-backed runtime can replace the heuristic later behind the same contract.

### Testing

- Use sqlite-backed tests where DB reads are involved.
- Keep CLI output structured JSON.
- Update `docs/project-tracker.md` after implementation.
