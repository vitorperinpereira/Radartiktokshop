## Story

**As a** backend engineer,
**I want** the core persistence schema and migration bootstrap wired into the project,
**so that** later ingestion, scoring, and reporting work can persist canonical product and run history safely.

## Acceptance Criteria

1. Shared SQLAlchemy metadata exists with naming conventions and exports for the persistence layer.
2. ORM models exist for the initial core tables: `products`, `product_aliases`, `product_snapshots`, `creators`, `product_signals`, `product_scores`, `content_angles`, `reports`, `ingestion_jobs`, and `pipeline_runs`.
3. Alembic is wired to the real metadata and an initial migration creates the base schema and key indexes.
4. The database helper layer can construct engines and session factories against the configured `DATABASE_URL`.
5. Tests exist for metadata registration and schema bootstrap behavior, with verification notes recorded if runtime dependencies are not installed locally.

## Dev Notes

### Implementation Context

- `docs/domain-model.md` defines the first table set and the main index strategy.
- `docs/setup.md` states PostgreSQL plus Alembic as the source of truth for persistence and requires UTC timestamps and explicit lineage.
- `docs/implementation-plan.md` places initial migrations and persistence ahead of ingestion adapters and scoring logic.
- Story 1.1 already created the project skeleton, `.env.example`, compose files, and base DB helper shell.

### Relevant Source Tree

- `services/shared/db`: persistence primitives, metadata, session helpers, and model exports
- `infra/migrations`: Alembic environment and revision history
- `tests/unit`: metadata and contract-level tests
- `tests/integration`: DB bootstrap tests

### Constraints

- Keep this story focused on schema and migration wiring only.
- Do not add ingestion adapters, normalization logic, or scoring logic here.
- Preserve portable model definitions where practical, while keeping PostgreSQL as the target runtime.

### Testing

- Prefer metadata and sqlite bootstrap tests for quick structural validation.
- If SQLAlchemy or Alembic are not installed locally, tests may use dependency-aware skips and the gap must be documented.
- Syntax compilation should still pass across `services`, `infra/migrations`, and `tests`.
