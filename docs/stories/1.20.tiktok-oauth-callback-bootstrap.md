## Story

**As a** local operator,  
**I want** a callback URL that can receive the TikTok Shop authorization code and persist the local token cache,  
**so that** I can complete TikTok OAuth locally without copying the code by hand every time.

## Acceptance Criteria

1. The TikTok auth bootstrap loads `TIKTOK_APP_KEY` and `TIKTOK_APP_SECRET` without requiring unrelated ingestion settings such as `APIFY_TOKEN`.
2. The FastAPI app exposes a local callback endpoint that accepts TikTok Shop authorization code query params, exchanges the code through the existing `TikTokOAuth` helper, persists the token cache, and returns a human-readable success or failure page.
3. Local operator docs and env examples document the callback setup clearly enough to register the redirect URL and run the local flow.
4. Automated tests cover auth settings loading plus callback success and validation failure behavior.

## Dev Notes

### Implementation Context

- `ingestion/auth/__main__.py` currently relies on `IngestionConfig.from_env()`, which pulls in `APIFY_TOKEN` even though the manual auth bootstrap only needs TikTok app credentials.
- `apps/api/main.py` is the current local HTTP surface and is the appropriate place for a callback endpoint that persists the existing token cache.
- `ingestion/auth/tiktok_oauth.py` and `ingestion/auth/token_cache.py` already define the exchange and cache behavior that the callback should reuse.

### Constraints

- Keep the OAuth exchange logic inside the existing TikTok auth helpers.
- Do not add dashboard-side control flow or couple the callback to the Streamlit UI.
- Keep the local flow aligned with the repository's CLI-first operating model.

### Testing

- Run `npm run lint`, `npm run typecheck`, and `npm test` before closing the story.
- Add focused tests for the auth env loader and the callback endpoint.
