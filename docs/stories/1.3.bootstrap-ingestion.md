## Story

**As a** project operator,
**I want** CLI-first ingestion from mock and CSV sources into the new persistence layer,
**so that** I can populate the product catalog and raw snapshots without relying on live platform connectors.

## Acceptance Criteria

1. A typed ingestion contract exists for source adapters and carries normalized candidate fields, source metadata, raw payload, and validation issues.
2. Mock and CSV adapters exist and emit the shared ingestion contract.
3. A persistence service creates `ingestion_jobs`, upserts canonical `products`, and writes `product_snapshots` from adapter records.
4. The CLI exposes commands to run database migration bootstrap and to ingest mock or CSV inputs end-to-end.
5. Tests cover normalization, adapter output, and sqlite-backed ingestion persistence behavior.

## Dev Notes

### Implementation Context

- `docs/source-governance.md` requires every source to flow through a typed adapter contract and to create an `ingestion_job`.
- `docs/domain-model.md` defines the persistence targets for this story: `products`, `product_snapshots`, and `ingestion_jobs`.
- `docs/implementation-plan.md` puts adapter-based ingestion ahead of scoring and reporting.
- Story 1.2 already delivered the schema, migration wiring, and DB session helpers.

### Relevant Source Tree

- `services/ingestion`: adapters, normalization, and service orchestration
- `services/shared/db`: persistence layer and ORM models
- `bin/radar.py`: CLI control surface
- `tests/unit` and `tests/integration`: contract and sqlite ingestion tests

### Constraints

- Keep this story focused on mock/CSV ingestion only.
- Do not add JSON adapters, scoring logic, or report generation here.
- Raw payload preservation is mandatory for every ingested snapshot.
- Canonical product upsert can be exact-key based for now; advanced alias strategy stays for later work.

### Testing

- Use sqlite-backed tests for persistence.
- Keep adapter tests deterministic and local-only.
- The CLI flow should remain usable via `python -m uv run radar ...`.
