## Story

**As a** product operator,  
**I want** the most visible runtime surfaces to use the aligned branding copy and naming rules,  
**so that** the app feels consistent with the branding book in the landing page, login page, dashboard empty states, and Streamlit root surface.

## Acceptance Criteria

1. The React landing and login pages use the branding-book-approved CTA and footer language.
2. Dashboard empty and error states use more actionable copy where the user currently gets only a description.
3. The Streamlit root page keeps the formal product name but presents the operator surface in a way that matches the current naming rules.

## Tasks / Subtasks

- [x] Identify runtime copy mismatches in the visible surfaces
- [x] Update landing and login page text to match the brand book
- [x] Tighten dashboard empty/error states for clearer next-step guidance
- [x] Update the Streamlit root page copy to better reflect the current formal naming and operator flow
- [x] Verify with the existing repo gates and update the story record
